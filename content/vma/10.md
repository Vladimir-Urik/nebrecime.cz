<h2 id="big-data">Big Data</h2>
<p><b>Big Data</b> označuje soubory dat, která jsou příliš velká nebo komplexní na zpracování tradičními nástroji. Definují ho tzv. <b>3V</b> (někdy 5V):</p>
<ul>
  <li><b>Volume (objem)</b> — petabyty a více (sociální sítě, IoT, logy)</li>
  <li><b>Velocity (rychlost)</b> — data vznikají a musí být zpracovávána v reálném čase</li>
  <li><b>Variety (různorodost)</b> — strukturovaná, polo-strukturovaná i nestrukturovaná data</li>
  <li><b>Veracity (věrohodnost)</b> — kvalita a přesnost dat</li>
  <li><b>Value (hodnota)</b> — užitečnost pro business</li>
</ul>
<p>Příklady: clickstream z webů, transakce bank, logy serverů, záznamy ze senzorů, příspěvky na sociálních sítích.</p>

<h2 id="nosql">NoSQL databáze</h2>
<p><b>NoSQL</b> (Not Only SQL) označuje databáze, které nespoléhají na relační model a pevné schéma. Vznikly pro potřeby Big Data a webových aplikací.</p>

<h3>Datové formáty v NoSQL</h3>
<table>
  <tr><th>Formát</th><th>Popis</th><th>Příklad</th></tr>
  <tr><td>JSON</td><td>Textový formát klíč-hodnota, hierarchický</td><td><code class="code">{"jmeno":"Jan"}</code></td></tr>
  <tr><td>BSON</td><td>Binární JSON (efektivnější, více typů)</td><td>Interní formát MongoDB</td></tr>
  <tr><td>XML</td><td>Značkovací jazyk, silně hierarchický</td><td><code class="code">&lt;jmeno&gt;Jan&lt;/jmeno&gt;</code></td></tr>
  <tr><td>CSV</td><td>Tabulková data oddělená čárkou</td><td><code class="code">Jan,20,Praha</code></td></tr>
</table>

<h3>Typy NoSQL databází</h3>
<table>
  <tr><th>Typ</th><th>Model dat</th><th>Příklady</th></tr>
  <tr><td>Dokumentové</td><td>JSON/BSON dokumenty</td><td>MongoDB, CouchDB</td></tr>
  <tr><td>Klíč–hodnota</td><td>Jednoduchý slovník</td><td>Redis, DynamoDB</td></tr>
  <tr><td>Sloupcové</td><td>Data uložena po sloupcích</td><td>Cassandra, HBase</td></tr>
  <tr><td>Grafové</td><td>Uzly a hrany (vztahy)</td><td>Neo4j, ArangoDB</td></tr>
</table>

<h3>Relační vs. NoSQL</h3>
<table>
  <tr><th></th><th>Relační (SQL)</th><th>NoSQL</th></tr>
  <tr><td>Schéma</td><td>Pevné (tabulky)</td><td>Flexibilní (dokumenty)</td></tr>
  <tr><td>Škálování</td><td>Vertikální (silnější server)</td><td>Horizontální (více serverů)</td></tr>
  <tr><td>ACID</td><td>Plná podpora</td><td>Omezená (BASE model)</td></tr>
  <tr><td>Vhodné pro</td><td>Strukturovaná data, složité dotazy</td><td>Big Data, rychlé zápisy, nestálé schéma</td></tr>
</table>

<h2 id="mongodb">MongoDB</h2>
<p><b>MongoDB</b> je dokumentová NoSQL databáze. Data ukládá ve formátu <b>BSON</b> (Binary JSON). Základní pojmy:</p>
<ul>
  <li><b>Databáze</b> — kontejner pro kolekce (≈ database v SQL)</li>
  <li><b>Kolekce</b> — skupina dokumentů (≈ tabulka v SQL)</li>
  <li><b>Dokument</b> — jeden záznam ve formátu JSON/BSON (≈ řádek v SQL)</li>
  <li><b>_id</b> — automaticky generovaný unikátní identifikátor dokumentu</li>
</ul>

<h3>Struktura dokumentu</h3>
<code>{
  <span class="g">"_id"</span>: <span class="r">ObjectId("...")</span>,
  <span class="g">"jmeno"</span>: <span class="r">"Jan Novák"</span>,
  <span class="g">"vek"</span>: <span class="r">25</span>,
  <span class="g">"adresa"</span>: {
    <span class="g">"mesto"</span>: <span class="r">"Praha"</span>,
    <span class="g">"psc"</span>: <span class="r">"110 00"</span>
  },
  <span class="g">"hobbies"</span>: [<span class="r">"programování"</span>, <span class="r">"šachy"</span>]
}</code>

<h2 id="crud">CRUD operace</h2>
<p>Čtyři základní operace s daty:</p>
<table>
  <tr><th>Operace</th><th>SQL</th><th>MongoDB</th></tr>
  <tr><td>Create</td><td>INSERT</td><td>insertOne / insertMany</td></tr>
  <tr><td>Read</td><td>SELECT</td><td>find / findOne</td></tr>
  <tr><td>Update</td><td>UPDATE</td><td>updateOne / updateMany</td></tr>
  <tr><td>Delete</td><td>DELETE</td><td>deleteOne / deleteMany</td></tr>
</table>

<code><span class="comment">// Vložení dokumentu</span>
db.uzivatele.insertOne({ jmeno: <span class="r">"Jana"</span>, vek: <span class="r">22</span> });

<span class="comment">// Výběr — všichni uživatelé starší 18</span>
db.uzivatele.find({ vek: { $gt: <span class="r">18</span> } });

<span class="comment">// Aktualizace — zvýšení věku o 1</span>
db.uzivatele.updateOne(
  { jmeno: <span class="r">"Jana"</span> },
  { $set: { vek: <span class="r">23</span> } }
);

<span class="comment">// Smazání</span>
db.uzivatele.deleteOne({ jmeno: <span class="r">"Jana"</span> });</code>

<h2 id="agregace">Agregace a MapReduce</h2>

<h3>Agregační pipeline</h3>
<p>Zpracování dokumentů ve více krocích (stages).</p>
<code>db.objednavky.aggregate([
  { $match: { stav: <span class="r">"zaplaceno"</span> } },      <span class="comment">// filtr</span>
  { $group: { _id: <span class="r">"$zakaznik"</span>,           <span class="comment">// seskupení</span>
              celkem: { $sum: <span class="r">"$castka"</span> } } },
  { $sort: { celkem: <span class="r">-1</span> } }              <span class="comment">// řazení</span>
]);</code>

<h3>MapReduce</h3>
<p>Programovací model pro zpracování velkých dat. Skládá se ze dvou fází:</p>
<ul>
  <li><b>Map</b> — každý dokument se transformuje na páry klíč–hodnota</li>
  <li><b>Reduce</b> — hodnoty se stejným klíčem se agregují</li>
</ul>
<code><span class="comment">// Map: vydej počet objednávek za zákazníka</span>
<span class="p">const</span> mapFn = <span class="p">function</span>() {
  emit(<span class="p">this</span>.zakaznik, <span class="r">1</span>);
};

<span class="comment">// Reduce: sečti</span>
<span class="p">const</span> reduceFn = <span class="p">function</span>(key, values) {
  <span class="p">return</span> Array.sum(values);
};</code>
<p>V moderním MongoDB je MapReduce nahrazeno efektivnější agregační pipeline.</p>
